# 用户管理交互修正设计

## 目标

修正账号系统第一版中侧边栏顺序和普通用户创建入口与预期不一致的问题。

## 侧边栏

- 管理员可见的“用户管理”排列在“设置”之前。
- “设置”始终是侧边栏最后一个菜单项。
- 普通用户仍不显示“用户管理”和“设置”。

## 创建用户

- 移除用户表格上方常驻的创建表单。
- 页面标题区域右侧显示“创建用户”按钮，账号角色仍固定为普通用户。
- 点击按钮打开创建弹窗。
- 弹窗包含用户名、密码和成人内容权限开关。
- 不增加确认密码字段；管理员可通过现有“重置密码”操作修正普通用户密码。
- 密码提示只显示“密码至少 8 个字符”；输入和后端仍保留 128 字符技术上限。
- 创建成功后关闭弹窗、清空表单并刷新用户列表。
- 创建失败时保留弹窗和输入内容，通过现有 toast 显示错误。

## 操作语义

- 普通用户的“停用”按钮使用红色危险语义；“启用”保持普通次级样式。
- “开启成人权限”和“关闭成人权限”统一使用橙色警告语义。

## 验证

- 测试管理员侧边栏中“设置”位于最后。
- 测试创建表单默认不显示，点击按钮后才出现弹窗。
- 测试弹窗字段、密码提示及创建参数。
- 测试创建成功后弹窗关闭并刷新列表。
- 运行前端全量测试和生产构建。
