# 初始管理员表单优化设计

## 目标

改善首次初始化管理员页面的信息层级和密码输入可靠性，不增加冗余说明，不修改后端账号接口。

## 页面设计

- “创建初始管理员”标题居中显示。
- 表单依次显示带可见标签的“用户名”“密码”“确认密码”输入框。
- 密码字段附近显示现有密码要求：至少 8 个字符。
- 不增加管理员权限、账号用途等额外介绍文案。
- 登录模式保持现有结构，不显示确认密码字段。

## 交互与校验

- 初始化提交前校验密码与确认密码一致。
- 两次密码不一致时不调用初始化接口，并显示明确的字段错误。
- 原有必填、密码最小 8 字符和最大 128 字符约束保持不变。
- 接口错误继续使用现有错误区域展示。

## 验证

- 测试初始化页面包含三个字段标签和密码要求提示。
- 测试密码不一致时不发起初始化请求并显示错误。
- 测试密码一致时仍按原流程创建管理员并登录。
- 运行前端相关测试和生产构建。
