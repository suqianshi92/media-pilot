# 账号系统第一版计划

本文记录账号系统讨论中已经确认的第一版边界。未确认的设计不提前写入。

## 产品边界

- Media Pilot 是单库多操作者系统，不提供多租户媒体库或目录隔离。
- Media Pilot 负责操作者身份、系统权限、任务归属和成人内容权限。
- Jellyfin 等媒体服务继续负责最终观看权限。

## 身份来源

- 第一版用户是由 Media Pilot 自身创建和管理的本地用户。
- 第一版不接入 Jellyfin 用户、OIDC、可信反向代理身份或其他外部账号系统。
- 外部登录方式如未来增加，应映射到 Media Pilot 用户，不改变本地用户作为系统身份主体的边界。

## 首次初始化

- 数据库不存在用户时，系统处于待初始化状态。
- 待初始化状态只开放初始化所需入口和健康检查，不开放业务 API。
- 初始化页面创建唯一的初始管理员；创建过程必须保证并发请求中只有一个成功。
- 初始管理员创建成功后，初始化入口关闭。
- 第一版不开放用户自行注册，后续用户只能由管理员创建。
- 不提供内置默认账号，也不通过长期环境变量保存管理员密码。
- 部署文档应提醒用户在可信网络中完成首次初始化，再将服务暴露给其他访问者。

## 登录会话

- 第一版使用数据库服务端会话，不使用 JWT。
- 浏览器通过 `HttpOnly` Cookie 持有随机会话 ID，不把访问令牌存入 `localStorage`。
- 用户会话有效期为 30 天；正常访问可以延长有效期。
- 会话按最后活跃时间滑动过期，但最多每 24 小时更新一次最后活跃时间和 Cookie，避免页面轮询持续写数据库。
- SSE 流式请求只在建立连接时计算一次活跃，不随每个流事件续期。
- 用户退出、被禁用或修改密码时，该用户的现有会话全部失效。
- 同一用户允许在多个浏览器或设备同时登录，每次登录创建独立会话。
- 普通退出登录只撤销当前会话，不影响其他设备。
- 管理员重置密码时撤销该用户全部会话。
- 第一版不提供设备列表、会话列表、单设备下线或独立强制退出功能。
- 会话 Cookie 始终设置 `HttpOnly`、`SameSite=Lax` 和 `Path=/`，不设置 `Domain`。
- HTTPS 请求或反向代理声明原始协议为 HTTPS 时设置 `Secure`；直接 HTTP 访问时不设置，以支持局域网初始化和访问。
- 第一版不强制 HTTPS，但部署文档应建议外部访问使用 HTTPS。

## CSRF 防护

- Cookie 登录配套统一 CSRF 防护，不能依赖各业务接口自行实现。
- 会话 Cookie 使用 `HttpOnly` 和 `SameSite=Lax`。
- 后端签发独立随机 CSRF Cookie，前端公共请求模块对所有修改状态的请求自动附加 `X-CSRF-Token`。
- 后端统一中间件校验 CSRF Cookie 与请求 Header；业务路由不重复编写校验逻辑。
- 登录、首次初始化和所有已登录写操作使用同一套 CSRF 机制。

## 角色模型

- 第一版只有 `admin`（管理员）和 `user`（普通用户）两种固定角色。
- 第一版不提供自定义角色或权限矩阵。
- 系统永久只有首次初始化创建的一个管理员账号。
- 后续创建的账号全部是普通用户，不能创建新的管理员，也不能调整用户角色。
- 唯一管理员可以管理用户、全局设置和全部任务。
- 普通用户可以使用媒体获取能力，但默认只能查看和操作自己发起的任务。
- 唯一管理员固定拥有成人内容权限；`can_access_adult` 只控制普通用户。

## 成人内容权限

- 唯一管理员固定通过成人内容权限检查；`can_access_adult` 是服务端对普通用户强制执行的权限，不只是前端可见性设置。
- 没有成人内容权限的用户进行全类型资源搜索时，不查询成人分类；显式成人搜索直接拒绝。
- 没有成人内容权限的用户不能使用 TPDB 成人元数据识别、成人元数据重搜、成人下载提交或成人任务操作。
- 成人任务及其详情、操作记录不能通过列表过滤或直接访问 ID 绕过权限。
- 内容发现对没有成人内容权限的用户不得主动推荐成人内容。
- 普通用户的成人内容权限被关闭后，其已有成人任务继续保留，但该用户立即不可见、不可操作；唯一管理员仍可处理。
- Prowlarr 查询按标准分类执行权限过滤；没有成人内容权限时，全类型搜索只查询电影和剧集分类，不查询成人分类。
- 后端对带有成人分类的返回结果再次过滤，但不承诺识别被资源站错误归入普通电影或剧集分类的成人资源。
- 第一版不增加 LLM 或标题关键词内容审查来弥补资源站错误分类。
- 下载任务和入库任务持久化 `is_adult` 标记，后端授权查询直接使用该字段过滤成人任务。
- 下载任务派生入库任务时必须继承任务归属和 `is_adult` 标记。
- 列表查询不得根据标题、文件名或路径临时推断成人任务。
- 成人资源搜索或 TPDB 成人元数据预选创建的下载任务，初始 `is_adult=true`。
- 普通电影、剧集搜索、手动上传和 watch 自动任务初始 `is_adult=false`。
- 后续确认使用 `tpdb_adult_movie` 或明确选择成人媒体库目标时，后端在继续处理前把任务改为成人任务。
- 管理员把错误成人元数据纠正为普通电影或剧集后，任务可以恢复为非成人任务。
- 客户端不能直接提交或修改 `is_adult`。

## 任务归属

- 用户提交的下载任务归属于当前用户。
- 下载任务派生的入库任务继承原下载任务的归属。
- 用户手动上传产生的入库任务归属于当前用户。
- watch 目录自动发现、定时同步等没有登录用户参与的任务归属于系统。
- 任务归属字段允许为空；无归属任务按系统任务处理。
- 账号系统上线前已经存在的历史任务不做归属回填，也不修改历史数据。
- 普通用户看不到系统任务，管理员可以查看和处理系统任务。
- 管理员处理其他用户的任务不会转移任务归属，也不增加特殊的“管理员代操作”展示。
- 第一版不记录每个任务步骤由哪一个具体用户执行；既有记录只需继续区分系统自动动作和人工动作。
- 任务责任人以任务归属为准，不从逐步操作记录推导。
- 普通用户直接向共享 watch 目录放置文件时，系统无法可靠识别文件来源用户，因此仍按系统任务处理。
- 成人任务在任务归属规则之上继续应用成人内容权限门禁。

## 自有任务权限

- 普通用户可以管理自己创建任务的完整生命周期。
- 普通用户可以暂停、继续、取消和删除自己的下载任务。
- 普通用户可以处理自己任务的元数据选择、Agent 决策和正常入库。
- 普通用户可以撤回自己任务发布的媒体库产物、删除该任务输入并删除任务。
- 普通用户不能覆盖已经存在的媒体库目标，包括目标来自其他任务或来源无法确认的情况。
- 覆盖既有媒体库目标只允许管理员执行。

## 用户生命周期

- 第一版不物理删除用户，只允许管理员停用和重新启用。
- 用户停用后不能登录，已有会话立即失效。
- 停用用户创建的下载任务和入库任务继续保留原任务归属。
- 管理员仍可查看和处理停用用户的任务。
- 用户重新启用后可以继续访问原有任务。
- 停用用户的用户名继续保留，不能由其他用户重复使用。
- 初始管理员是受保护账号，永久保持已启用的管理员身份，不能被停用或降级。
- 初始管理员可以正常修改密码，也可以通过紧急恢复流程重置密码。
- 系统不提供管理员角色变更，因此不存在停用、降级或新增管理员的操作入口。

## 初始管理员密码恢复

- 忘记初始管理员密码时，通过服务器终端执行专用容器命令重置，不提供匿名网页找回、邮箱找回或安全问题。
- 恢复命令自动定位唯一的初始管理员，不要求用户记住其用户名。
- 新密码在交互式终端中隐藏输入并确认，不通过命令参数、环境变量或日志传递。
- 密码重置成功后撤销初始管理员的全部现有会话，不改变用户名、角色、成人内容权限或任务归属。
- 正常运行时支持通过 `docker compose exec` 执行；主容器无法启动时应提供对应的一次性 `docker compose run` 恢复方式。
- 实现该命令后，必须在部署与排障文档中记录准确、可直接执行的两种命令，不能只保留在本计划中。

## 常规密码管理

- 用户知道原密码时，可以自行修改密码；修改后撤销该用户的现有会话。
- 普通用户忘记密码时，由管理员在用户管理页直接设置一个新密码。
- 管理员重置密码后撤销该用户的现有会话，用户随后使用新密码正常登录。
- 第一版不增加临时密码、首次登录强制改密、邮箱或短信找回流程。

## 用户资料

- 第一版用户只包含用户名、密码哈希、角色、成人内容权限、启用状态、创建时间和更新时间。
- 用户名同时用于登录和界面展示，按大小写不敏感规则保持唯一。
- 用户名创建后不能修改。
- 系统不保存明文密码。
- 第一版不增加邮箱、手机号、昵称、头像或个人简介。

## 用户创建默认值

- 首次初始化创建系统中唯一的 `admin`；后续新用户固定为 `user`，不提供角色选择。
- 新用户默认处于启用状态。
- 唯一管理员固定拥有成人内容权限，不显示成人权限开关。
- 普通用户的成人内容权限默认关闭。
- 创建普通用户时，管理员可以主动打开成人内容权限。
- 用户创建后，唯一管理员可以继续调整普通用户的成人内容权限和启用状态。

## 密码规则

- 第一版密码长度为 8 至 128 个字符。
- 允许中文、英文、数字、符号和空格，不强制大小写、数字或特殊符号组合。
- 密码按用户完整输入处理，不自动去除首尾空格，也不截断。
- 不要求用户定期修改密码。
- 首次初始化、管理员重置和用户自行修改密码使用同一规则。

## 第一版不做

- 不对登录失败进行计数、限速、验证码验证或账号锁定。

## 匿名访问

- 系统尚未初始化时，未登录用户进入初始管理员创建页面。
- 系统完成初始化后，未登录访问应用页面时统一跳转到登录页。
- 登录成功后进入现有应用内容，不提供未登录只读浏览模式。
- 匿名访问只开放前端静态文件、登录、初始化状态、待初始化时的初始管理员创建，以及不泄露配置详情的基础健康检查。
- 首页、任务、资源搜索、内容发现、下载、手动上传、文件资产、流式接口、应用设置和后台状态全部要求登录。

## 登录跳转

- 未登录访问应用页面时，登录页保留原目标地址。
- 登录成功后返回原目标地址；没有目标地址时进入首页。
- 会话过期收到 `401` 时跳转登录页并保留当前地址。
- 重新登录后仍无权访问原目标时，按后端 `403` 或 `404` 正常展示，不循环跳转。
- 初始管理员创建成功后直接建立登录会话并进入首页，不要求再次登录。

## 设置可见性

- 完整全局设置、连接状态和用户管理只允许管理员读取和修改。
- 普通用户界面不显示系统设置和用户管理入口。
- 普通业务页面需要的已启用元数据档案等信息，通过精简的可用能力响应提供。
- 可用能力响应不包含 API Key、服务地址、目录路径、限速配置或其他管理信息。
- 普通用户的个人菜单只提供修改自己的密码和退出登录。

## 用户管理页面

- 侧边栏增加唯一管理员可见的“用户管理”入口。
- 用户管理使用单页表格，不增加用户详情页或卡片布局。
- 表格显示用户名、角色、启用状态、成人内容权限和创建时间。
- 复用任务列表页的 `DataTable`、服务端分页和横向滚动逻辑，不另建表格实现。
- 初始管理员行显示管理员身份，但不提供停用、角色或成人权限操作。
- 普通用户行提供启用开关、成人内容开关和重置密码操作。
- 页面顶部提供“创建用户”按钮，点击后在弹窗中填写用户名、密码和成人内容权限；角色固定为普通用户。
- 创建普通用户不要求二次确认密码，界面只提示密码至少 8 个字符；输错后可使用既有重置密码操作修正。

## 后端数据授权

- 管理员首页和任务统计可以覆盖全部授权可见任务；普通用户首页、任务列表和统计只包含自己的任务。
- 后台 Agent 状态卡及其诊断接口只允许管理员访问。
- 任务归属和成人内容权限过滤必须在后端接口及数据库查询中执行，不能先返回全量数据再由前端过滤。
- 列表分页、排序、筛选和 `total` 必须基于当前用户授权后的数据集合计算。
- 任务详情、文件资产、Agent 消息、人工决策及所有写操作必须在后端校验对象访问权限。
- 前端只负责隐藏无权限入口和展示后端已经授权的数据，不承担权限安全边界。

## 任务归属展示

- 管理员任务列表增加创建者列；用户任务显示用户名，自动任务和历史无归属任务显示“系统”。
- 停用用户的任务继续显示原用户名。
- 管理员查看任务工作台和下载详情时，在基础信息中显示创建者。
- 普通用户列表和详情不显示创建者字段，因为其可见任务均归属于自己。
- 第一版不提供按用户筛选任务，也不从用户管理页跳转到指定用户的任务列表。

## 授权错误响应

- 未登录访问受保护接口返回 `401`，前端转到登录页。
- 已登录用户访问管理员专属功能返回 `403`。
- 用户通过 ID 访问其他用户、系统或无成人权限的任务及其子资源时返回 `404`，不泄露对象是否存在。
- 用户可以查看自己的任务但无权执行管理员专属动作时返回 `403`。

## 统一认证与授权结构

- 使用 FastAPI 原生 `Depends` 和 `Annotated` 声明接口所需的当前用户、管理员身份、任务访问范围或已授权对象，不自定义权限装饰器。
- 同组接口可以在 `APIRouter` 上统一要求登录；需要当前用户或管理员对象的接口通过类型依赖取得结果。
- 列表接口的依赖返回当前用户的数据访问范围，而不只是允许或拒绝请求。
- 任务访问范围至少包含当前用户 ID、是否可查看全部任务和成人内容权限；授权查询 module 把该范围转换为数据库过滤条件。
- 普通用户的列表查询限制为自己的任务并叠加成人内容权限过滤；唯一管理员固定通过成人权限检查，可以查询全部用户和系统任务。
- 过滤条件必须先于排序、分页和 `total` 计算执行。
- 单任务、下载、人工决策、文件资产和候选句柄通过统一的已授权对象解析 module 找到所属任务并校验权限。
- 后台 worker、下载同步和 Agent 内部流程继续使用不带登录用户过滤的原始 Repository，避免破坏系统自动任务。

## 流式连接授权

- Agent 和内容发现 SSE 在建立连接时验证登录状态及对象权限。
- 流持续期间最多每 30 秒重新检查一次会话和权限。
- 用户退出、被停用或成人权限关闭后，相关流最迟 30 秒内断开。
- 关闭 Agent 观察流不取消或回滚后台 Agent 运行。
- 前端主动退出登录时立即中止当前浏览器中的流。
- 权限复核由统一流式授权 module 提供，不在各 SSE 接口重复实现。

## 资源候选句柄

- 资源搜索产生的候选句柄继续保存在进程内缓存中，有效期保持 30 分钟，不写入数据库。
- 候选缓存项记录创建用户 ID 和成人任务标记。
- 元数据识别和下载提交只能由创建该候选的同一用户执行，管理员也不跨用户复用候选句柄。
- 使用候选时重新检查用户当前的成人内容权限，不能只依赖创建候选时的权限。
- 无权访问或已经过期的候选统一按不存在处理。

## 浏览器状态隔离

- 内容发现消息继续使用 `sessionStorage`，不写入数据库。
- 内容发现存储键包含当前用户 ID，不同用户不能读取或沿用彼此的会话消息。
- 用户退出登录时清除该用户的内容发现消息；登录身份变化时重新加载对应用户的状态。
- 主题和语言继续作为浏览器级非敏感偏好共享，不按用户隔离。

## 实施范围

- 新增本地用户、服务端会话、登录初始化和用户管理能力。
- 为下载任务和入库任务增加可空任务归属及成人任务标记，不回填历史任务。
- 使用 FastAPI 依赖、授权查询 module 和已授权对象解析 module 统一接入现有接口。
- 调整 flow read model，使授权过滤先于排序、分页和统计。
- 增加登录、初始化、用户管理和个人密码修改界面，并按角色调整现有导航和任务归属展示。
- 不触及生产媒体文件、下载器内容或媒体库目录；数据库变化保持增量兼容。

## 验证

- 待初始化、登录、退出、会话续期、会话撤销和管理员终端密码恢复符合本文规则。
- 匿名、普通用户和唯一管理员对页面及接口的 `401 / 403 / 404` 行为正确。
- 普通用户的任务列表、统计、详情和写操作只作用于自己的授权任务，分页与 `total` 不泄露全局数据。
- 系统任务只对管理员可见，下载任务派生入库任务时正确继承归属和成人标记。
- 成人搜索、元数据、任务和内容发现按普通用户权限过滤，管理员固定拥有成人能力。
- 普通用户不能覆盖既有媒体库目标，但可以管理自己任务的其他完整生命周期。
- 用户停用、密码修改和权限变更正确撤销会话与流式访问。
- 候选句柄、内容发现浏览器状态和文件资产不能跨用户访问。
- 既有无归属历史任务继续正常运行并只对管理员可见。
