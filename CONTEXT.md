# Media Pilot

Media Pilot 是一个自托管媒体获取与入库自动化系统。本文只保留当前代码和产品语言需要共享的核心词汇；历史迁移、旧流程和一次性决策不放在这里。

## 核心词汇

**媒体获取流程**:
用户视角的一次完整媒体处理过程，从资源发现开始，经过下载、识别、元数据匹配、发布和必要的人工决策。

**下载任务**:
Media Pilot 持久化跟踪的下载过程。它表示资源候选已提交给下载器后的传输状态，不等同于入库任务。

**下载器全局限速**:
Media Pilot 对专用下载器实例施加的全局上传 / 下载速度上限。它作用于整个 qBittorrent 实例，不按单个下载任务或种子区分；零值表示不限速。

**入库任务**:
对一个下载完成或外部导入的媒体输入进行识别、元数据匹配、发布和清理的业务对象。

**任务工作台**:
单个入库任务的查看、纠正、确认和继续处理页面。它同时展示任务事实和 Agent 对话。

**任务事实**:
围绕入库任务形成的结构化、可复查信息，例如源文件、当前元数据、候选、集文件映射、发布计划、写入结果和操作记录。

**Agent 对话**:
绑定到单个入库任务的持久化交互历史，用于解释判断、接收用户纠正、展示工具调用摘要和承载人工决策。

**Agent 运行**:
围绕单个入库任务推进的一次可恢复 Agent 执行过程。同一入库任务同一时间最多应有一个活跃 Agent 运行。

**Agent 工具**:
可审计的 Media Pilot 业务能力入口。Agent、API 和测试应尽量通过同一受控入口调用业务能力。

**人工决策请求**:
Agent 或系统在无法安全自动推进时向用户提出的持久化问题，可包含结构化选项和必要的自由文本入口。

**等待用户**:
入库任务因存在未完成的人工决策请求而暂停自动推进的用户可见状态。

**受控发布动作**:
对媒体库写入、覆盖、撤回或清理等有文件系统影响的操作。必须通过 Agent 工具或既有后端能力执行并留下审计。

**任务输入节点**:
某个入库任务实际占用的源文件或源目录。它不能泛化为 downloads 根目录、watch 根目录、媒体库目录或任意父目录。

**发布对象**:
一次入库实际写入媒体库的媒体内容集合。它由已确认的主媒体输入和已确认归属的字幕或附属文件组成，不等同于原始任务输入节点。

**当前发布上下文**:
系统用于判断当前媒体库发布产物是否仍有效的结构化数据，例如发布计划、写入结果和文件资产。

**重发布源**:
当原任务输入已被清理但当前发布产物仍存在时，用于纠正重发布的受控来源。它只能来自该任务自己的当前发布上下文，不能泛化为媒体库中的任意文件。

**事件时间线**:
围绕入库任务发生过的可审计历史事件，例如复制、发布、撤回、删除输入和失败记录。

**元数据自动确认阈值**:
系统配置中用于判定候选是否足够明确、可自动进入发布流程的置信度和候选差距规则。

**元数据状态**:
入库任务对当前发布产物元数据完整度的任务事实，例如未知、完整或无元数据。它不是任务流程状态，不能替代入库成功、等待用户或 Agent 失败等状态。

**安全硬门禁**:
不可由 Agent 放宽的安全边界，例如路径越界、目标覆盖、复杂多主视频和高风险输入类型。

**语义判断软门槛**:
Agent 基于任务事实、工具结果和系统阈值对媒体类型、候选正确性和字幕归属等语义问题做出的可解释判断。

**复杂电影输入**:
看起来仍属于电影入库，但包含多视频、复杂目录、样片花絮或字幕归属不明确等需要人工选择的任务输入。

**BDMV 电影目录**:
已解包的蓝光电影目录输入，任务以该目录为入库单元。常见结构通常包含 `BDMV/`（含 `index.bdmv`/`STREAM/`）和可选的同级目录（如 `CERTIFICATE/`）；发布目标是完整电影目录。

**ISO 镜像输入**:
`.iso`/`.img` 文件不作为入库来源，复杂输入判定中直接视为不支持类型，不进入电影重建发布链路。

**剧集入库**:
围绕分季分集作品进行识别、元数据匹配、集文件映射和发布的入库处理。

**集文件映射**:
把一个或多个剧集视频文件稳定对应到具体 season 和 episode 的任务事实。发布剧集时应以它为准，不应在写入阶段重新猜测。

**绝对集数映射**:
资源文件使用全局 episode 编号，而元数据提供方使用可覆盖这些编号的 season 表达同一作品时，把文件编号映射为该 season 下 episode 编号的集文件映射方式。

**成人影片库根**:
用于发布成人影片元数据档案结果的独立媒体库根目录。缺失配置时不应自动 fallback 到常规电影库根。

**元数据候选选择**:
当元数据候选未达到自动确认条件时，由用户在人工决策请求中选择一个候选作为任务元数据事实。

**人工辅助检索**:
用户在任务工作台中手动搜索并指定元数据候选，用于把用户确定的元数据选择写入任务事实。

**通用 Agent 输入**:
用户在没有 pending 人工决策请求时，主动向 Agent 对话发送自然语言要求，例如纠正元数据、要求重新搜索或说明当前任务判断错误。

**元数据纠正**:
用户发现当前刮削结果错误后，通过 Agent 对话触发重新搜索、替换候选、重建发布计划或重新发布。

**元数据重发布**:
用户为已入库任务重新选择元数据后，系统基于可用源重新生成媒体库产物的纠正过程。它同时覆盖错误元数据纠正和无元数据入库后的补元数据。

**无元数据入库**:
当系统无法找到可信元数据时，用户确认后仍把任务输入发布到媒体库的入库方式。它只能绕过元数据缺失或不明确，不能绕过安全硬门禁。
_Avoid_: 强行入库

**撤回入库**:
删除某个入库任务已经发布到媒体库的产物，使任务从已发布状态退回到可重新处理或可结束的状态。

**撤回并删除任务输入**:
高危操作，表示撤回已发布产物后继续删除入库任务关联的源文件或源目录，并结束该入库任务。

**源文件清理策略**:
Agent 主线下控制任务输入清理是否保留、询问、移入回收区或进入删除确认的用户配置语义。

**目标冲突决策**:
发布目标路径已存在时向用户提出的人工决策，用于选择覆盖发布目标或取消本次处理。

**剧集发布冲突**:
剧集发布时围绕具体 season 或 episode 产物发生的目标冲突。已存在的 show 根目录本身不是冲突。

## 不变量

- 媒体获取流程列表是只读视图，不表示新的持久化父实体。
- 下载器全局限速是应用配置对专用下载器实例的期望状态；保存配置时应尝试同步到下载器，启动时在期望值尚未成功同步过时补同步。
- 入库任务拥有任务工作台；任务工作台同时展示任务事实与 Agent 对话。
- Agent 工具是业务能力入口，Agent 运行只是调用者之一。
- 有 pending 人工决策请求时，入库任务进入等待用户。
- 用户回复人工决策请求后应产生可读的系统动作消息，不应伪装成用户自然语言消息。
- Agent 流式观察只是 Agent 运行的观察通道；连接断开不应取消或回滚运行。
- 后台 Agent 状态只用于诊断和首页提示，不替代入库任务、下载任务、Agent 运行或事件时间线。
- 受控发布动作必须通过 Agent 工具或既有受控后端能力执行。
- 撤回并删除任务输入只能作用于具体任务输入节点，不能作用于受控根目录。
- 源文件清理失败不应把已经完成入库的任务改为入库失败。
- 无元数据入库应复用已经确认的发布对象，不应把整个原始任务输入节点无差别复制到媒体库。
- 无元数据入库后的纠正重发布可以使用当前发布产物作为重发布源；不得先删除唯一可用源，再尝试重新发布。
- 元数据自动确认阈值决定候选是否可自动推进；未达到阈值时应转为人工决策请求。
- 无元数据入库只能绕过元数据缺失或不明确，不能绕过路径、输入结构、目标冲突等安全硬门禁。
- 安全硬门禁不能被 Agent 运行放宽；语义判断软门槛可以由 Agent 基于工具事实解释和决策。
- BDMV 电影目录的发布目标是完整电影目录，覆盖与冲突判断都按目录粒度处理，不允许将其混入单文件电影发布语义。
- 复杂电影输入的决策选项应由后端基于任务事实标准化生成，不应让 Agent 自行拼接文件路径或安全相关 payload。
- `.iso`/`.img` 输入一律视为不支持输入类型，不能进入电影发布路径或产物记录链路。
- 剧集入库应使用独立的剧集发布 Agent 工具，不应把电影发布工具泛化成模糊工具。
- 标准 `SxxEyy` 编号优先于绝对集数映射；目录名中的 season 字样不能单独决定绝对集数映射的目标 season。
- 剧集发布冲突应按具体 season 或 episode 产物判断；show 根目录可复用。
- 成人影片库根由最终确认的元数据档案决定；`tpdb_adult_movie` 不应发布到常规电影库根，除非用户显式配置为同一路径。

## 易混点

- “Agent 模式”不是一次不可恢复的长 LLM 调用；本项目中是围绕入库任务的可恢复 Agent 运行。
- “任务详情”优先称为任务工作台，因为它不是只读信息页。
- “工具”默认指内部受控 Agent 工具，不是 MCP。
- “剧集支持”不是独立规则式链路，而是 Agent 主线中的剧集入库能力。
- “任务内自由对话”不是全站 AI 助手；上下文绑定单个入库任务。
- “智能资源搜索”属于资源发现能力，不等同于入库任务 Agent 主线。
